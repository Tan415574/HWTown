❓ Questions

Frequently asked

How does the 14-day free trial work? ↓
You get full access to all features for 14 days with no credit card required. At the end of the trial you can subscribe for R120/month or your account pauses — no charges, no surprises.
Can I add more than one child? ↓
Yes — one family subscription covers all children in the household. Each child gets their own town, coins, and progress tracking from the shared parent account.
Is it safe for young children? ↓
Homework Town is designed for ages 6–12. There is no external messaging, no social features, no ads, and no in-app purchases. Children can only interact within your family account.
Do children need their own device? ↓
No — Homework Town runs in any browser on any device. A parent can hand a child their phone or tablet, the child does their tasks, and the parent gets the notification on their own device.
What age is it best suited for? ↓
Homework Town is designed for children aged 6 to 12. The game mechanics, themes, and reading level are all aimed at primary school children.
How do I cancel? ↓
Cancel any time from your account settings with one click. You keep access until the end of the billing period and are never charged again.

🔧 Setup Guide
Get Homework Town fully working in a few steps.
🔒 Step 1 — Fix Firestore Security Rules
Children log in without Firebase Auth, so your Firestore rules need to allow their reads. Without this the Reward Shop and some features won't load for kids.
1
Go to console.firebase.google.com → select your homeworktown project
2
Click Firestore Database in the left menu, then click the Rules tab at the top
3
Replace everything in the editor with the rules below, then click Publish
rules_version = '2'; service cloud.firestore { match /databases/{database}/documents { match /parents/{uid} { allow read, write: if request.auth != null && request.auth.uid == uid; } match /children/{childId} { allow get, list: if true; allow write: if request.auth != null || resource == null; allow update: if true; } match /homework/{docId} { allow get, list: if true; allow write: if request.auth != null; } match /rewards/{docId} { allow get, list: if true; allow create: if request.auth != null; allow update, delete: if request.auth != null && request.auth.uid == resource.data.parentId; } match /rewardPurchases/{docId} { allow read: if request.auth != null; allow create: if true; allow update: if request.auth != null; allow delete: if request.auth != null; } match /bonusTasks/{docId} { allow get, list: if true; allow write: if request.auth != null; } match /taskProgress/{docId} { allow read, write: if true; } } }
🎁 Step 2 — Sync Rewards to Children
Do this once after setting up rules, and whenever you add new rewards

The Reward Shop reads from a cached copy stored on each child's account. Press the button below to push your current rewards to all children. Adding new rewards from the Rewards tab does this automatically.
